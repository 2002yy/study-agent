# README v0.6.3

## 默认入口调整

v0.6.3 将系统默认入口改为 `微信群`。

- 打开页面后，主区默认进入微信群聊天面板。
- 原有 `单人主讲` 模式仍然保留，只是改成可切换入口。
- 入口配置由 `memory/internal_state.md` 中的 `entry_mode` 控制。

支持两个值：

- `wechat`
- `single`

字段缺失时，系统默认按 `wechat` 处理。

## 入口切换

侧栏新增 `聊天入口` 选择：

- 微信群
- 单人主讲

切换后会写回 `memory/internal_state.md`，下次打开页面仍会保持当前入口。

## 微信群模式行为

- 微信群模式下，主区默认显示微信群面板。
- 用户发送消息后，才会触发群聊互动。
- 群聊互动调用 `generate_interactive_wechat_reply`。
- 每个新群聊线程里，用户第一次发言会触发轻微惊讶。
- 同一群聊线程内不会重复惊讶。
- 打开页面不会自动调用 LLM。
- 不会自动生成每日开场白。

## 单聊模式行为

- 单聊模式仍保留完整主聊天链路。
- 入口切到 `single` 后，主区显示单人主讲聊天面板。
- 不影响现有路由、LLM、memory、safe_mode、memory_mode 逻辑。

## 状态条变化

状态条现在会显示入口信息：

- 入口：微信群 / 单人主讲

微信群模式下优先展示：

- 入口
- 群聊状态
- 未读数量
- 互动氛围
- 性能模式

单人主讲模式下优先展示：

- 入口
- 当前角色
- 当前模式
- 当前模型
- 性能模式

## 手动验收

1. 打开页面，确认默认主区进入微信群聊天，而不是单聊区。
2. 查看侧栏，确认 `聊天入口` 默认是 `微信群`。
3. 切换到 `单人主讲`，确认主区切换为单聊面板。
4. 再切回 `微信群`，确认主区恢复为微信群面板。
5. 在微信群模式下只打开页面，不发送消息，确认不会自动触发群聊回复。
6. 在一个新群聊线程里发送第一条消息，确认会触发轻微惊讶。
7. 在同一线程继续发送第二条消息，确认不会再次触发首次惊讶。
8. 查看 `memory/internal_state.md`，确认 `entry_mode` 会随着切换写回为 `wechat` 或 `single`。
