# 个人学习 Agent 系统 v0.6 视觉增强说明

## 1. 版本定位

v0.6 是视觉增强版本。

本版本不改变 v0.1–v0.5.x 已经完成的核心逻辑，而是在现有功能稳定的基础上，增强界面观感、角色识别度、微信群沉浸感和长期陪伴感。

v0.6 的核心目标是：

> 让系统从"功能型学习 Agent"升级为"有角色感、有群聊感、有状态感的学习伙伴系统"。

本版本重点包括：

- 角色头像
- 微信群气泡 UI
- 未读红点
- 角色状态条
- 群聊顶部栏
- 背景与主题资源
- 视觉资源目录规范
- 长文本、代码块、Markdown 表格的可读性保护

---

## 2. 本版本不改变的内容

v0.6 不重写以下核心逻辑：

- LLM API 调用逻辑
- Flash / Pro 双模型逻辑
- 自动路由逻辑
- 长期记忆读取与写入逻辑
- 课后更新机制
- 微信群记忆候选机制
- 内部模式系统
- safe_writer 安全写入机制
- session_logger 日志机制

v0.6 只做视觉层增强，不改变系统行为边界。

说明：

- `backgrounds/` 存放聊天背景
- `icons/` 存放未读、模型、记忆等小图标
- 如果用户没有放入真实图片，系统应使用 `default.png` 或文字 fallback
- 不允许因为缺少图片导致页面崩溃

### 3.2 当前实现说明

v0.6 已完成的视觉模块采用轻量实现：

- **头像**：采用文字 fallback（"7"/"晴"/"妲"/"萤"），不依赖真实图片；详见 `src/ui/avatar.py`
- **状态条**：页面顶部状态栏，显示角色/模式/模型/氛围/阶段；详见 `src/ui/status_bar.py`
- **微信群气泡**：卡片式布局，彩色昵称 + 头像 + 气泡边框；详见 `src/ui/wechat_bubble.py`
- **全局 CSS**：暗色 Catppuccin 主题，详见 `src/ui/theme.py`
- **未读红点**：侧栏"微信群"旁粉红徽标
- **群聊顶部栏**：卡片内"课后学习小组 — 时间"
- **时间分割线**：`——` 行自动渲染
- **代码块保护**：`overflow-wrap` + `white-space: pre-wrap`
- `assets/` 目录保留为后续真实图片资源入口，当前无 .png/.webp 文件

### 3.3 实际文件结构

```
src/ui/

```

## 4. 视觉资源规范

### 4.1 头像规范

推荐尺寸：128×128 / 256×256
推荐格式：png / webp

命名规则：
- `march7.png`
- `keqing.png`
- `nahida.png`
- `firefly.png`
- `default.png`

角色映射：
- march7 → 三月七
- keqing → 刻晴
- nahida → 纳西妲
- firefly → 流萤

要求：
- 头像缺失时使用默认头像
- 不允许图片加载失败导致页面报错
- 头像只用于本地个人学习系统
- 如果未来公开发布，应替换为原创头像或授权素材

### 4.2 背景规范

背景优先级：
1. 不影响阅读
2. 不抢文字
3. 低对比、浅色、柔和
4. 支持关闭背景

推荐风格：浅色微信背景 / 柔和学习桌面背景 / 低饱和二次元背景 / 简洁纯色背景

禁止：高饱和背景 / 复杂人物背景 / 强纹理背景 / 影响代码块阅读的背景

### 4.3 气泡规范

微信群消息应支持：头像 / 角色名 / 消息气泡 / 时间 / 线程信息 / 未读状态

基础格式：
```
[头像] 三月七
  今天这轮其实挺关键的……
```

角色颜色建议只做轻微区分：
- 三月七：活泼、明亮
- 刻晴：清晰、利落
- 纳西妲：柔和、安静
- 流萤：温暖、低饱和

注意：
- 不要用强烈颜色压过正文
- 长代码块不应被气泡宽度压坏
- Markdown 表格、列表、代码块要保持可读

## 5. v0.6 功能范围

### 5.1 角色头像显示

- 主聊天区、微信群区、角色状态条中能显示角色头像
- 用户选择角色时显示对应头像
- 自动路由时显示最终选中的角色头像
- 群聊中每条角色消息前显示对应头像
- 头像缺失时使用默认头像

### 5.2 微信群气泡 UI

将 Markdown 风格微信群升级为类群聊卡片显示：
- 每条消息显示角色名
- 每条消息显示头像
- 每条消息使用气泡包裹
- 支持用户消息显示
- 支持长文本折叠或正常换行

### 5.3 未读红点

当 `chat/wechat_unread.md` 中存在未读内容时，侧栏显示未读提示：
- 清空未读后红点消失
- 不清空 `wechat_group.md`
- 重启应用后仍能根据 `wechat_unread.md` 判断未读状态

### 5.4 角色状态条

主界面顶部显示当前状态：
- 当前角色 / 当前模式 / 当前模型
- 互动氛围 / 微信群状态 / Memory 状态
- 自动路由后显示最终结果
- 手动选择时显示手动选择结果

### 5.5 群聊顶部栏

微信群区域顶部显示：
- 课后学习小组
- 成员：三月七、刻晴、纳西妲、流萤
- 状态：未读 / 可互动 / 当前线程

### 5.6 背景与主题

- 背景可关闭
- 背景不影响代码块
- 背景不影响 Markdown 表格
- 不因背景图片缺失报错

## 7. v0.6 验收用例

### 测试 1：头像 fallback

删除某个角色头像 → 重启 → 页面不崩溃，该角色显示 default 头像或文字 fallback。

### 测试 2：角色状态条

选自动角色/自动模式/自动模型 → 输入"CNN 为什么要参数共享？" → 状态条显示 纳西妲/苏格拉底/Pro。

### 测试 3：微信群气泡

生成微信群反馈 → 查看未读 → 四角色分别显示头像、昵称、气泡，长文本正常换行。

### 测试 4：未读红点

生成反馈 → 侧栏出现红点 → 清空未读 → 红点消失 → `wechat_group.md` 不被清空。

### 测试 5：代码块可读性

让 Agent 输出 Python 代码 → 代码块不被气泡压坏，缩进正常，背景不影响阅读。

### 测试 6：Markdown 表格可读性

让 Agent 输出 Markdown 表格 → 表格正常显示，不被气泡样式破坏。

### 测试 7：背景关闭

关闭背景或使用默认主题 → 页面恢复简洁背景，文字可读性正常。

## 10. 风险与注意事项

### 10.1 版权与素材

如果使用现有角色头像，仅限本地个人学习使用。如果未来公开分享、开源或发布，应替换为原创或授权素材。

### 10.2 不要影响可读性

本系统主要用途仍然是学习、论文修改、项目推进、代码分析、课后复盘。视觉增强不能影响正文阅读。

### 10.3 不要让视觉层改业务逻辑

v0.6 的 UI 不应该改变 memory 写入权限、safe_mode、relationship_mode 边界、router 选择、wechat_state 状态机、after_session 写入流程。视觉层只负责展示。
