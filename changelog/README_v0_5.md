# v0.5 验收文档

## 1. v0.5 目标

实现关键字匹配自动路由：根据用户输入自动判断角色、教学模式和模型档位。手动选择优先于自动判断。

## 2. 新增文件

| 文件 | 说明 |
|------|------|
| `src/router.py` | 关键字匹配路由器，`python -m src.router` 可自测 |

## 3. 修改文件

| 文件 | 改动 |
|------|------|
| `app.py` | 侧栏角色/模式/模型增加"自动"选项（默认）；用户输入后调用 `route_request()` 解析；主区显示"本轮路由结果"expander；日志传入 `route_info` |
| `src/session_logger.py` | `log()` 新增 `route_info` 参数；`save()` 输出路由详情（选择/解析/覆盖/原因） |

## 4. 自动路由规则表

| 用户输入特征 | 角色 | 模式 | 模型 | 说明 |
|-------------|------|------|------|------|
| 为什么/本质/底层/机制/原理/从根上讲 | nahida | 苏格拉底 | pro | 追问本质→深度引导 |
| 怎么办/下一步/实现/代码/bug/报错/方案 | keqing | 项目 | pro | 任务/排错→收束边界 |
| 我来讲/你听我解释/我来解释/帮你检查 | nahida | 费曼 | pro | 用户复述→找漏洞 |
| 论文/摘要/降AI/章节/参考文献/表述 | keqing | 论文 | pro | 论文→审视结构 |
| 累/复盘/收尾/今天到这/陪我理一下 | firefly | 普通 | flash | 疲惫→轻陪伴 |
| 入门/开始学/先问我/别直接讲/新手 | march7 | 苏格拉底 | flash | 启动→引导入门 |
| 无匹配 | nahida | 普通 | flash | 默认 |

## 5. 手动覆盖规则

- 用户手动选择的项（非"自动"）**直接使用**，不参与匹配
- 只有值为 `"auto"` 的项才由路由器决定
- 可以混合：手动角色 + 自动模式 + 自动模型

## 6. 验收用例

| # | 步骤 | 预期 |
|---|------|------|
| 1 | 侧栏全部选"自动"，问"CNN 为什么要参数共享？" | 路由→nahida/苏格拉底/pro；expander 显示原因 |
| 2 | 问"这个 bug 下一步怎么修？" | 路由→keqing/项目/pro |
| 3 | 问"我来解释一下卷积，你帮我检查" | 路由→nahida/费曼/pro |
| 4 | 手动选择三月七+苏格拉底+自动，问"CNN 本质" | 角色/模式保留手动，模型自动→pro；覆盖=是 |
| 5 | 问"hello" | 路由→nahida/普通/flash（默认） |
| 6 | 保存日志，检查 `logs/sessions/` | 含完整路由信息（选择/解析/覆盖/原因） |
| 7 | `python -m src.router` | 5 个测试用例全部命中 |
