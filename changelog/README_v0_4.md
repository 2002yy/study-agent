# v0.4 验收文档

## 1. v0.4 目标

实现微信群机制：课后自动生成四人角色群聊反馈，写入未读消息和完整群聊存档。用户可随时查看、清空未读。

## 2. 新增文件

| 文件 | 说明 |
|------|------|
| `src/wechat.py` | 群聊生成、读取、追加、清空 |
| `templates/wechat_update.md` | 群聊生成 prompt 模板（四人风格、边界、格式） |
| `chat/wechat_group.md` | 完整群聊存档（累计追加，不删） |
| `chat/wechat_unread.md` | 未读消息缓存（查看后清空） |

## 3. 修改文件

| 文件 | 改动 |
|------|------|
| `app.py` | 主区增加"生成微信群反馈"按钮（确认写入后出现）；侧栏增加"微信群"区域（查看未读/清空未读）；导入 wechat 模块 |
| `src/session_logger.py` | 新增 `set_wechat_status()`、`set_wechat_unread_cleared()`；日志 header 记录 `微信群反馈: generated/none` 和 `微信群未读: cleared` |

## 4. 使用流程

### 4.1 完整流程

```
对话
  → [生成课后更新预览]（侧栏）
    → 5 expander 预览
      → [确认写入长期记忆]
        → [生成微信群反馈] ← v0.4 按钮
          → 写入 wechat_unread.md + wechat_group.md
          → 主区展示群聊内容
```

### 4.2 查看未读消息

1. 点击侧栏 **微信群 → 查看未读消息**
2. 主区出现包含 `st.container` 的群聊展示区，格式：

```
【三月七】
今天聊了...

【刻晴】
下一步不要跳步...

【纳西妲】
本轮本质是...

【流萤】
辛苦了...
```

### 4.3 清空未读消息

1. 点击侧栏 **微信群 → 清空未读消息**
2. `wechat_unread.md` 重置为占位内容
3. `wechat_group.md` 不受影响

### 4.4 检查完整存档

打开 `chat/wechat_group.md`，确认每次生成的群聊均已追加，原文不丢失。

## 5. 验收用例

| # | 步骤 | 预期 |
|---|------|------|
| 1 | 正常对话数轮 | 正常 |
| 2 | 点"生成课后更新预览" | 5 expander 出现 |
| 3 | 点"确认写入长期记忆" | st.success，下方出现"生成微信群反馈"按钮 |
| 4 | 点"生成微信群反馈" | st.success + 主区展示四人 `【角色】` 格式群聊 |
| 5 | 检查 `chat/wechat_unread.md` | 内容为刚生成的群聊 |
| 6 | 检查 `chat/wechat_group.md` | 末尾追加了相同群聊 |
| 7 | 点"查看未读消息" | 主区展示群聊内容 |
| 8 | 点"清空未读消息" | st.success，`wechat_unread.md` 清空为占位 |
| 9 | 检查 `chat/wechat_group.md` | 存档完整，未被清空 |
| 10 | 点"结束本轮并保存日志" | `logs/sessions/` 下日志含 `微信群反馈: generated` / `微信未读: cleared` |
| 11 | 新一轮对话不点微信群按钮 | 保存后日志含 `微信群反馈: none` |
