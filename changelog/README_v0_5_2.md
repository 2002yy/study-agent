# v0.5.2 角色深度与微信群互动补丁

## 1. 版本定位

v0.5.2 解决两个早期遗留问题：

1. **角色过于简单** — v0.1 的角色 prompt 只有几句话设定，随着系统复杂度提升，角色变得扁平、容易串味
2. **微信群单向** — v0.4 的微信群只是"生成→查看→清空"的广播模式，缺乏真正的伙伴互动感

本版本不做新主功能，是对已有角色的深度加固和微信群交互能力的升级。

## 2. 角色深度升级

四个角色文件从 ~170 字升级到 ~1000-1100 字，统一 9 段结构：

| 章节 | 内容 |
|------|------|
| 1. 核心定位 | 一句话定义角色在整个系统中的唯一价值 |
| 2. 性格层次 | 表层表现 + 深层动机 + 面对用户时的关注点 |
| 3. 教学风格 | 怎么讲概念、怎么引导思考 |
| 4. 项目推进风格 | 在项目/任务场景中的行为 |
| 5. 论文修改风格 | 在论文审读场景中的行为 |
| 6. 微信群风格 | 在群聊中的发言定位 |
| 7. 互动方式 | 与其他三个角色的配合关系 |
| 8. 禁止跑偏 | 明确的负面行为边界 |
| 9. 动态记录区 | 运行时持续更新的观察和印象 |

### 角色差异强化

| 角色 | 核心定位 | 不是 |
|------|----------|------|
| 三月七 | 学习启动器 | 不是搞笑角色 |
| 刻晴 | 任务收束器 | 不是压力源 |
| 纳西妲 | 概念结构化器 | 不是玄学总结器 |
| 流萤 | 课后陪伴者 | 不是主讲老师 |

## 3. 微信群互动升级

### 核心设定

- 四个角色默认不知道用户能看到微信群（她们以为是私下讨论区）
- 用户第一次在群里发言时，她们会有轻微惊讶反应（**仅一次**）
- 惊讶之后，微信群从"课后反馈区"进入"可互动群聊"

### 状态管理

新增 `chat/wechat_state.md`：

```yaml
user_has_joined_group: false → true (用户首次发言后)
first_join_reaction_done: false → true (惊讶反应触发后)
mode: unread_feedback → interactive_group
```

### 交互流程

```
用户查看未读 → 在群里说一句 → 发送
  → 追加到 wechat_group.md
  → 检查 wechat_state
  → 首次：四人惊讶反应
  → 之后：正常互动回复
  → 追加回复到 wechat_unread.md + wechat_group.md
```

## 5. 验收用例

| # | 步骤 | 预期 |
|---|------|------|
| 1 | 查看 `roles/march7.md` | 含 9 段结构，字数 > 800 |
| 2 | 查看 `roles/keqing.md` | 含"任务收束器"定位 |
| 3 | 课后更新 → 生成微信群反馈 → 查看未读 | 正常显示四人消息 |
| 4 | 在输入框输入"我看得到你们在说什么"→ 发送 | 四人惊讶反应（仅一次） |
| 5 | 再次在群聊发言 | 正常互动，不重复惊讶 |
| 6 | 检查 `chat/wechat_state.md` | `user_has_joined: true`, `mode: interactive_group` |
| 7 | 检查 `chat/wechat_group.md` | 含用户消息和角色回复 |
| 8 | 清空未读后检查 `wechat_group.md` | 群聊历史保留 |
| 9 | 保存日志后检查 | 含 `微信群互动: generated` |

## 6. 边界约束

- 不做 v0.6 视觉增强
- 不做陪伴强度开关
- 不做恋爱模式
- 不破坏 v0.1-v0.5.1 已有功能
- 不改 API 双模型链路
- 首次惊讶仅一次，不重复
- 不展开无关剧情
- 不编造用户情绪
